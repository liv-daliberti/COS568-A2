\section{Discussion}
The old CPU numbers in this repo came from a one-node debug launcher. The rerun above uses true multi-node CPU and GPU jobs with 1 rank per node. GPU training is still faster end-to-end: the new GPU speedups over CPU are 2.36x for Task 2(a), 4.25x for Task 2(b), and 4.88x for Task 3.

CPU Task 2(b) is 28.21\% faster than CPU Task 2(a), and CPU DDP is another 16.06\% faster than CPU all-reduce on the same 4-node allocation.

For Task 4, CPU DDP has an average communication-overhead percentage of 64.471\% versus 50.260\% for manual all-reduce, a relative reduction of -28.276\%. On GPU, the same trace-derived metric is 89.858\% for DDP versus 84.125\% for all-reduce, so the relative reduction is -6.815\%. Even when the GPU trace metric is not lower for DDP, DDP still wins on end-to-end iteration time.
